Validation Studio

Validation Studio is an integrated debugging and verification space for parsing and validating industrial payloads against rulesets or schema specs.

Validation Engine Architecture
Raw Message Payload
↓
Validation Studio In-Memory Router
↓
JSON Schema / XSD Validation
↓
Monaco-rendered Markdown Report

Supported Structures & Rules

Format Type Validation Strategy Diagnostics Detail
XML + XSD Validates structures against W3C standards using schema locations. Displays line and column tags on schema mismatches.
JSON Schema Checks draft-07 formats, including regex, limits, and type maps. Details exact object pathways failing validation.
SWIFT MT Standard checks vs Enhanced JSON-defined custom rule evaluations. Lists block issues or flags validation policy violations.
CSV + CSVW Validates column formats, required entries, and unique columns. Lists record numbers containing failures.

SWIFT Custom Enhanced Rules Example

Customize validation checks by editing the Schema Editor with rule definitions:

{
  "enhanced": true,
  "refPrefix": "EXB-",
  "amountLimit": 5000000.0,
  "restrictedCurrencies": ["RUB", "IRR"],
  "highRiskJurisdictions": ["Iran", "Tehran"]
}

Crypto Studio

Crypto Studio performs secure AES-256-GCM authenticated encryption on configuration secrets, database properties, and application variables.

Cryptographic Profile

  • Encryption Algorithm: AES-256-GCM (Galois/Counter Mode).
  • Key Derivation Function: PBKDF2WithHmacSHA256 (65,536 iterations).
  • Metadata: Prefixes a 16-byte random salt and a 12-byte initialization vector (IV) directly to the ciphertext.

Note: AES-256-GCM provides built-in integrity verification via a 128-bit authentication tag, protecting your secrets against alteration.

How to Decrypt Secrets in the IDE

To decrypt encrypted settings inside the workspace:

  1. Select Edit -> Decrypt Ciphertext... from the top toolbar.
  2. Input your secret decryption key in the password prompt.
  3. Paste the Base64 ciphertext into the payload text editor.
  4. Click Decrypt Payload to review raw plaintext configurations.

Camel XML Integration Example

<bean id="cryptoProcessor" class="com.routebuilder.kameletstudio.crypto.FieldCryptoProcessor">
    <property name="algorithm" value="AES-256-GCM"/>
    <property name="fields" value="ssn,creditCardNumber"/>
</bean>

Map Studio (MAP)

Map Studio allows visual schema matching and payload mapping. These mappings are stored in the global registry file `validation-mapping.json`.

Visual Mapping Controls

The visual mapper renders structural representations of source structures and schema definitions side-by-side:

  • Field Connection: Drag a node from the left Source Tree and drop it onto the right Target Schema to establish a link.
  • Visual Feedback: Displays active mappings in real-time using dynamic connection lines.
  • Interactive Controls: Right-click on trees to add or remove links.

`validation-mapping.json` Registry Example

{
  "mappings": [
    {
      "name": "Customer XML to JSON Schema",
      "messagePath": "messages/customer.xml",
      "schemaPath": "schemas/customer-schema.xsd",
      "type": "XML + XSD"
    }
  ]
}

Transform Studio

Transform Studio designs and tests message transformations across CSV, XML, JSON, and flat file formats.

Transformation Engines

Engine DSL Component Primary Purpose
Smooks smooks://config.xml High-performance mapping and Java bean binding.
FreeMarker freemarker://template.ftl Template-based XML, JSON, or text generation.
JSLT jslt://transform.jslt JSON-to-JSON structure mappings.
Groovy groovy://script.groovy Scripted message payload mapping.

Route Integration Snippet

Reference your transformation assets inside your Camel YAML DSL integration files:

- from:
    uri: "direct:start"
    steps:
      - to: "smooks:file:mappings/smooks-config.xml"
      - to: "mock:result"

Diagram Studio

Diagram Studio visualizes Apache Camel routes from XML or YAML files in real-time using Mermaid.js.

Key Features

  • Dynamic Flowchart Renders: Renders route flowcharts automatically as you type.
  • Endpoint Visuals: Displays distinct icons representing external systems, database endpoints (sql:), message queues (jms:), and web services.
  • EIP Flow Representation: Visually renders content-based routers, splitters, aggregators, and filters as routing branches.

Faker Studio

Faker Studio creates mock datasets to simulate realistic transaction workloads.

Generator Functions

Inject fake data fields using double-brace notation in your mock templates:

  • {{name.fullName}} - Generates random full names.
  • {{internet.email}} - Generates synthetic email addresses.
  • {{finance.iban}} / {{finance.bic}} - Generates bank identifiers.
  • {{random.number(min, max)}} - Generates integers within a specified range.

Continuous Background Simulation

Faker Studio can run simulations in the background, writing generated messages to your local directories or target endpoints at a configured delivery rate (e.g., 5 files/second).

Export Studio

Export Studio compiles, packages, and exports your Camel routes and database migration configurations for production deployment.

1. Liquibase Database Migration

Generates Liquibase-compliant XML changelogs (changelog.xml) to automate database migrations across test and production environments.

<changeSet id="1" author="routebuilder">
    <createTable tableName="CUSTOMERS">
        <column name="ID" type="numeric">
            <constraints primaryKey="true" nullable="false"/>
        </column>
        <column name="NAME" type="varchar(100)"/>
    </createTable>
</changeSet>

2. SQL Schema Export (Oracle & Postgres)

Exports database schemas as standard SQL scripts tailored to your target database dialect.

Oracle Dialect DDL Example:

CREATE TABLE CUSTOMERS (
    ID NUMBER PRIMARY KEY,
    NAME VARCHAR2(100) NOT NULL,
    STATUS VARCHAR2(20) DEFAULT 'ACTIVE'
);

PostgreSQL Dialect DDL Example:

CREATE TABLE IF NOT EXISTS customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    status VARCHAR(20) DEFAULT 'ACTIVE'
);

3. File System Structure Export

Creates standardized directory layouts to organize runtime binaries, configuration properties, and file routing targets.
